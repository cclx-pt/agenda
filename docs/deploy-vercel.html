Passos do deploy

Da importação do repositório ao site público.

1

Importar o repositório

Vercel → Add New… → Project → importar cclx-pt/agenda (branch main), equipa CCLX.

2

Nome de projeto único

Se o botão Deploy estiver cinzento, é porque já existe um projeto agenda. Muda o nome (ex.: agenda-cclx) ou abre o projeto existente e faz Redeploy.

3

Application Preset → Other

O vercel.json usa builds + routes, por isso escolhe Other e deixa Build/Output/Install Command vazios. Root Directory = ./.

4

Environment Variables

Expande Environment Variables → Import .env → seleciona .env.vercel. Preenche os <...> a partir de server/.env. Aplica a Production.

5

Deploy

Carrega Deploy. O Vercel constrói o frontend para dist/ e empacota o backend como função serverless (server/serverless.js).

6

Definir o CORS_ORIGIN real

Depois do 1.º deploy, copia o URL final (ex.: https://agenda-cclx.vercel.app) para a variável CORS_ORIGIN e faz Redeploy.

7

Tornar público opcional

Se aparecer a página de login do Vercel, desliga Settings → Deployment Protection → Vercel Authentication para o domínio de produção ficar acessível a todos.
